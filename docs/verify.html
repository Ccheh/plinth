Plinth's design is multi-underwriter on purpose: each posts independently on chain; investors choose whose lens to weight.

About this demo. The Aster trade data is fetched once at submission time (mainnet API requires private-key signing, so we can't live-call from your browser without exposing keys). The reconciliation math runs entirely in your browser when you press the button — there's no server computing the answer. The Plinth side is read live from Arc Testnet RPC.

Why "Aster"? Aster L1 was picked for v0 because its trade history is publicly readable via an API tied to an on-chain delegated signer — a perfect target for verifier-style underwriting. The exact same pattern works for any future public-chain perp DEX, including Arc-native ones once they launch. Aster is a demo target, not a dependency.
