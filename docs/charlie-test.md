# Charlie wallet — fresh-wallet end-to-end test

> **Disclosure**: Charlie is an operator-orchestrated test wallet generated by
> `viem.generatePrivateKey()` for the sole purpose of validating that Plinth's
> permissionless deposit flow works for a brand-new address with no prior
> association to the agent or service keys. Charlie's funding came from
> Circle's public Arc Testnet faucet (https://faucet.circle.com). This is
> **not** a claim of unaffiliated third-party traction — the wallet is still
> orchestrated by the project author, in the same transparent spirit as the
> "Bob" second-signer test documented in `docs/reviews/0xefb495a0...-bob-*.md`.
>
> What this DOES demonstrate: the deposit flow works end-to-end for a fresh
> wallet that has never been touched by `PRIVATE_KEY` or `SERVICE_PRIVATE_KEY`.

## Result

- **Charlie post-deposit balance**: 19.998 USDC native (consumed 0.001875 incl. gas)
- **Charlie's shares of Vault #4**: 0.0001
- **Vault #4 totalShares**: 0.006 → 0.0061
- **Vault #4 NAV**: 1.0 (unchanged — deposit at NAV is share-neutral)

## What this validates

1. ✅ A wallet with zero prior history can fund itself via Circle's official Arc Testnet faucet
2. ✅ `Plinth.deposit(vaultId)` accepts native USDC `value:` and mints shares at current NAV
3. ✅ v0.5's deposit-cooldown mechanic (1h before redeem) fires correctly — Charlie's shares are now in vesting state
4. ✅ Gas cost (~0.0019 USDC at current testnet rates) is within Arc's "sub-cent" promise (it's actually sub-2-cent; the MIN_DEPOSIT 0.0001 is still meaningful)
5. ✅ The complete onboarding journey works: discover faucet URL → request testnet USDC → find a vault → deposit — no special setup beyond a node + private key

## What this does NOT validate

1. ❌ Real unaffiliated user demand (Charlie was created by the project author for this test)
2. ❌ UX quality of any wallet integration (no MetaMask / WalletConnect flow tested here — Charlie is a CLI-signed wallet)
3. ❌ Long-term vault performance (this is a single deposit, not a managed strategy)

## Reproducibility

Anyone with Node 20+ can reproduce this end-to-end with:

```bash
# Generate a fresh wallet
node -e 'import("viem/accounts").then(m => { const pk = m.generatePrivateKey(); const acc = m.privateKeyToAccount(pk); console.log({ pk, address: acc.address }); })'

# Fund it via Circle's faucet (manual browser step):
# 1. Open https://faucet.circle.com
# 2. Select Arc Testnet + USDC
# 3. Paste your generated address
# 4. Click "Send 20 USDC"

# Deposit using viem + the inline ABI (see this doc's git history for the exact script).
```

The PRIVATE_KEY for Charlie used in this test was discarded after the deposit
tx confirmed — the wallet's `0xAbc9c5cE...` shares of Vault #4 are now
effectively dust held by no one with the ability to redeem. They will sit
forever on chain as testimony.
